Addendum

Project Process and Creative Decisions

Concept Development: This project's goal was to design an interactive, story-based game into HTML, CSS, and JavaScript. It was interesting to hear the idea of a "cyber detective" storyline in a futuristic city because creativity could be explored in the design, there would be interactive choices, and the opportunity to add mini-games was there, making the project challenging and exciting at once.

Planning and Structure: I planned out the plot structure and various paths and endings before ever writing a line of code. I opted for a "Choose Your Own Adventure" format in which there are various stages leading to choices whose outcome determines the final ending. The stage is created as choice points where every choice led to a new stage or mini-game, and therefore having multiple endings due to the action of the user.

UI and Aesthetic Choices: Neon color values and glow effects in teal, pink, and dark background colors were used to immerse the viewers within an aesthetic of cyberpunk. The help of CSS animations animated the buttons, and pulsing activity on the click gave a futuristic feel to the effect. I opt for the aesthetic with the sense of a technologically advanced, digital world that would give the feel of being a cyber detective.

Interactivity and Game Mechanics: The game logic was managed with JavaScript. This code dynamically changed the stages and added interactivity to the user interface. The main things were:

  • Dynamic Story Stages: For each episode of the novels, an object was chosen to represent the text, choices, and consequences. Such an approach was highly flexible, allowing for expansion or modification of storyline content.
  • Mini-games: Simple puzzle challenges were interspersed to add richness to the game. For example, for decryption, the user had to input a code to continue, simulating the process of investigation.
  • Audio Elements: Background music and sound effects were used to build a more immersive environment. The music toggle helped users control the audio to suit their taste.

Responsiveness: I made sure the game was quite responsive, that it would work amazingly well across different devices. I used the CSS grid as well as media queries to ensure layout change so things worked correctly with buttons and content across both small and big screens.

Addendum Modal: As I needed to fulfill the needs of the project and be able to have a reflection space, I added an "Addendum" button, opening the modal. It then gives insight and challenges faced, thereby giving a professional feel to the project. This was my chance to explore using modal windows to take the user interface forward.

Challenges and Solutions

Story Stages Management

Challenge: It was really difficult to link multi-stage stories together initially because any choice had to lead into another stage or mini-game suitably.

Solution: I organized the stages in a JavaScript object, so now it becomes even easier to keep track of them: keys for each clearly defined stage. This structure also maintains modular and manageable code.

Audio Auto-play Restrictions:

Challenge: Audio autoplay is disabled by default in modern browsers and breaks the immersive experience.

Solution: I included a flag to start music only after the first user interaction, such as clicking a button. This was a workaround and meant that the music played in line with browser policies.

Ensuring Uniform Design:

Challenge: The consistency of colors, style of fonts, and animation throughout the elements was critical to giving it a professional look.

Solution: Replaced the use of primary colors by CSS variables, applied common animations to all buttons, and then cross-tested across all sizes of screens to be a design element.

Modal functionality:

Challenge: Add interactivity to the Addendum modal and ensure it closes properly when clicking out of the modal area.

Solution: I added some JavaScript to close the modal when clicking outside the content. That makes it feel a lot more professional and easy to use.

Insights and Lessons Learned

This was an excellent opportunity to synthesize storytelling, design, and technical skills. It taught me how to plan and structure my code, especially where there is a branching storyline. CSS will also have a powerful role in adding visual interest to the experiences without getting too complicated. Finally, this project helped drive home the importance of design choices as both functional and fitting for the thematic goals of the project. In effect, building this game was highly exciting and really built my confidence in the handling of JavaScript for interactive applications and the use of CSS to bring life to designs.

Good luck in your adventure!
