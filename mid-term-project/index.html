Addendum

Project Process and Creative Decisions

Concept Development: This project's goal was to design an interactive, story-based game into HTML, CSS, and JavaScript. It was interesting to hear the idea of a "cyber detective" storyline in a futuristic city because creativity could be explored in the design, there would be interactive choices, and the opportunity to add mini-games was there, making the project challenging and exciting at once.

Planning and Structure: I planned out the plot structure and various paths and endings before ever writing a line of code. I opted for a "Choose Your Own Adventure" format in which there are various stages leading to choices whose outcome determines the final ending. The stage is created as choice points where every choice led to a new stage or mini-game, and therefore having multiple endings due to the action of the user.

UI and Aesthetic Choices: Neon color values and glow effects in teal, pink, and dark background colors were used to immerse the viewers within an aesthetic of cyberpunk. The help of CSS animations animated the buttons, and pulsing activity on the click gave a futuristic feel to the effect. I opted for the aesthetic with the sense of a technologically advanced, digital world that would give the feel of being a cyber detective.

Interactivity and Game Mechanics: The game logic was managed with JavaScript. This code dynamically changed the stages and added interactivity to the user interface. The main elements were:

  • Dynamic Story Stages: Each stage was represented by an object with text, choices, and consequences. This approach was highly flexible, allowing for easy expansion or modification of the storyline.
  • Mini-games: Simple puzzle challenges were included to add depth to the game. For example, the decryption mini-game required the user to input a code to continue, simulating an investigative process.
  • Audio Elements: Background music and sound effects were added to enhance immersion. A toggle allowed users to control the audio to suit their preferences.

Responsiveness: I ensured that the game was fully responsive, working smoothly across devices. CSS Grid and media queries were used to adjust the layout, ensuring that buttons and content displayed properly on both small and large screens.

Addendum Modal: To fulfill the project requirements and provide a space for reflection, I added an "Addendum" button that opens a modal. This feature allowed me to explore modal windows and enhance the user interface by providing insights and challenges faced during the project.

Challenges and Solutions

Story Stages Management

Challenge: Initially, linking multi-stage stories was complex, as each choice had to lead to another stage or mini-game appropriately.

Solution: I organized the stages in a JavaScript object, with clearly defined keys for each stage. This structure kept the code modular and manageable.

Audio Autoplay Restrictions:

Challenge: Modern browsers block autoplay for audio, which interrupted the immersive experience.

Solution: I added a flag to start music only after the first user interaction, such as clicking a button. This workaround allowed music to play in compliance with browser policies.

Ensuring Uniform Design:

Challenge: Maintaining consistent colors, font styling, and animations across all elements was essential for a professional look.

Solution: I used CSS variables for primary colors and applied common animations to all buttons. Regular testing across screen sizes ensured consistent design elements.

Modal Functionality:

Challenge: Making the Addendum modal interactive and ensuring it closed correctly when clicking outside the modal area.

Solution: I added JavaScript to close the modal when clicking outside its content area, improving usability and giving a professional touch to the project.

Insights and Lessons Learned

This project provided a valuable opportunity to combine storytelling, design, and technical skills. It taught me the importance of planning and structuring code, especially when dealing with branching storylines. CSS also played a powerful role in creating visually engaging experiences without overwhelming complexity. This project reinforced the importance of making design choices that are both functional and aligned with the thematic goals of the project. Overall, building this game was a rewarding experience, and I gained confidence in handling JavaScript for interactive applications and using CSS to bring designs to life.

Good luck in your adventure!
